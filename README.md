# Cartoon Rendering

OpenCV 이미지 처리 기술을 이용해 사진을 만화 스타일로 변환하는 Python 프로그램입니다.

## 동작 원리

세 가지 핵심 이미지 처리 단계를 조합합니다.

1. **색 평탄화** — `cv2.bilateralFilter`를 10회 반복 적용해 텍스처를 부드럽게 만들면서 경계선은 유지, 만화 특유의 단색 영역을 만들어냄
2. **채도 강조** — HSV 색공간으로 변환 후 채도(S 채널)를 2.2배 증폭, 만화·애니 특유의 선명하고 진한 색감 표현
3. **윤곽선 추출** — `cv2.adaptiveThreshold`(세밀한 선)와 `cv2.Canny` + dilation(굵은 외곽선)을 결합해 만화풍의 두꺼운 검정 윤곽선 생성

## 데모

### Good 예시 — 단순한 배경의 피규어 사진

흰색/회색 단색 배경에 피사체가 명확히 구분되는 피규어 이미지입니다.

잘 되는 이유:
- 배경이 균일해 피사체와 배경의 경계가 명확함
- 피사체의 색 영역이 넓고 단순해 색 평탄화 효과가 뚜렷함
- 강한 조명으로 선명한 경계선이 형성되어 윤곽선 추출이 깔끔하게 됨

배경과 피사체의 경계가 불분명하고 복잡한 텍스처를 가진 우주 사진입니다.

잘 안 되는 이유:
- 전경/배경 구분이 없어 이미지 전체에 노이즈성 윤곽선이 생성됨
- 수많은 별이 화면 전체에서 잘못된 엣지를 감지하게 만듦
- 어두운 배경으로 인해 마스킹이 과하게 적용되어 검정 영역이 넓어짐

## 한계점

- **복잡한 배경** — 배경에 디테일이 많은 경우(풍경, 밤하늘, 군중 등) 엣지 검출기가 노이즈를 과도하게 감지해 지저분한 결과가 나옴
- **부드러운 색 전환 손실** — 양방향 필터가 얼굴 음영, 옷의 주름 등 형태를 표현하는 부드러운 그라데이션을 평탄화해버림
- **세밀한 디테일 소실** — 텍스트, 가는 머리카락, 복잡한 패턴 등 작은 요소들이 색 평탄화 과정에서 사라짐
- **피사체 인식 불가** — 모든 픽셀을 동등하게 처리하므로 주요 피사체와 배경을 구별하지 못해 중요한 영역을 보호할 수 없음
- **고해상도 이미지에서 느림** — 높은 sigma 값으로 양방향 필터를 10회 반복 적용하므로 고해상도 이미지는 처리 시간이 오래 걸림
